Context
The Problem
Before any analysis is written, engineers spend hours collecting data manually from disconnected systems.
⏱️

Manual Correlation

Engineers pull logs from Splunk or Datadog, cross-reference Jira defects, and read code diffs before writing a single line of analysis.

📉

Inconsistent Quality

Analysis depth varies by engineer experience. Junior engineers miss signals experienced ones would catch — creating uneven incident response.

📤

Sharing Overhead

The same findings get re-written multiple times — once for engineers, once for leadership, once for customers — at a cost of hours per incident.

Goal: Given an incident and its metadata, automatically query the right data sources in parallel, synthesise findings into a structured report, and let the engineer review, chat, edit, and share — all from one UI.

Flows
Incident Intake
Engineers submit manually via the UI form, or incidents are auto-ingested from Jira and other systems via webhooks. Both paths run the same pipeline.

🖥️ Path A — Manual (UI Form)

1
Engineer fills title, description, severity, metadata
2
POST /api/analysis returns job_id immediately
3
UI polls every 3 s while analysis runs in the background
4
Report appears in UI on completion

⚡ Path B — Automatic (Webhook)

1
Jira creates or transitions a ticket matching trigger rules
2
Jira fires webhook → HMAC signature validated
3
Adapter maps payload to Incident model → Celery task enqueued
4
Slack / email notification sent with link: /analysis/{id}
Trigger rules are config-driven in config.yaml — only specific issue types, priorities, and project keys fire analysis, keeping noise low.

Capabilities
Key Features
Beyond automated analysis — a complete engineer review workflow with AI collaboration built in.
🤖

Multi-Agent Pipeline

Orchestrator LLM decides which agents to activate. Log, code, and defect agents run in parallel via LangGraph's Send API. Synthesiser produces root cause, workaround, and prioritised actions with a confidence score.

💬

Chat for Clarifications

Engineers ask follow-up questions with full incident context — raw sub-agent findings, current report, and conversation history all in the system prompt. Responses stream in real time via SSE. AI can propose direct field edits (accept / dismiss).

📝

Analysis Versioning

Every edit — manual or chat-suggested — creates a new version. The original AI output (v0) is never overwritten. Version history with diffs is available in the UI. Share and export always use the latest approved version.

📈

Token Usage Tracking

Each analysis records per-agent token counts and estimated cost. Aggregate trends identify which agents are most expensive, enabling targeted prompt optimisation over time.

🔌

Pluggable Adapters

Every external system sits behind an abstract base class. Add Splunk, Datadog, Salesforce, or any other source by subclassing one file and adding one config.yaml entry — no pipeline changes.

🔬

Evaluation Agent

On-demand, post-closure agent that scores the prior AI analysis against the actual resolution. Emits prompt-improvement hints written back to the database for continuous analysis quality improvement.
